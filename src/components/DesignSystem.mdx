## Usage Guidelines

### Color Usage

- Use semantic color tokens for consistent theming
- Primary colors for main actions and brand elements
- Muted colors for secondary content and backgrounds
- Destructive colors only for error states and dangerous actions

### Typography Guidelines

- Use the base size (1rem) for body text
- Headings should use larger sizes (xl, 2xl, 3xl)
- Small text (sm, xs) for metadata and captions
- Maintain consistent line heights for readability

### Spacing Consistency

- Use multiples of 4px (spacing scale) for consistent rhythm
- Apply spacing systematically: 4px, 8px, 12px, 16px, 24px, 32px
- Maintain consistent margins and padding across components

### Responsive Design

- Mobile-first approach using our breakpoint system
- Test components across all breakpoints
- Ensure touch targets are at least 44px for mobile devices

## Dark Mode Support

All design tokens automatically adapt to dark mode using CSS custom properties. The system seamlessly switches between light and dark themes while maintaining contrast ratios and accessibility standards.

```css
/* Light mode (default) */
:root {
  --background: 0 0% 100%;
  --foreground: 240 10% 3.9%;
  /* ... */
}

/* Dark mode */
.dark {
  --background: 240 10% 3.9%;
  --foreground: 0 0% 98%;
  /* ... */
}
```
